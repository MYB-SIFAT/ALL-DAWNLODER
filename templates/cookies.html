01

Cookie File Method

Place .txt cookie files in the Cookies/ folder

1
Export cookies from your browser

Install the "Get cookies.txt LOCALLY" or "Cookie-Editor" browser extension. Log in to YouTube / Facebook / etc., then export cookies as a Netscape format .txt file.

2
Name the file correctly

Save the file with the platform name inside the Cookies/ folder:

YT Cookies/youtube.txt
FB Cookies/facebook.txt
IG Cookies/instagram.txt
TT Cookies/tiktok.txt
TW Cookies/twitter.txt
ALL Cookies/cookies.txt (all platforms)
3
Restart the app

Click the Run button to restart. The app picks up cookie files automatically on each download.

Cookie files are read per-download. You can replace them at any time without restarting.
02

Environment Variable Method

Paste a Base64-encoded cookie string into Replit Secrets

1
Export your cookies as a .txt file

Same as Method 1 — use a browser extension to export the Netscape cookie file for the platform you want.

2
Convert to Base64

Run this command in your terminal (or use an online Base64 encoder):

base64 -w 0 cookies.txt

On Windows (PowerShell):

[Convert]::ToBase64String([IO.File]::ReadAllBytes("cookies.txt"))
3
Add to Replit Secrets

Open the Secrets tab in your Replit project and create a secret with the name below, pasting your Base64 string as the value:

PlatformSecret Name
YTYOUTUBE_COOKIES
FBFACEBOOK_COOKIES
IGINSTAGRAM_COOKIES
TTTIKTOK_COOKIES
TWTWITTER_COOKIES
ALLCOOKIES
4
Restart the app

After adding a secret, click Run to restart. The app will automatically decode and use the cookie on every download.

Recommended for Replit. Secrets stay encrypted and are not stored in your codebase.
Priority Order

When multiple cookies are available, the app uses: ① Platform env var → ② Generic COOKIES env var → ③ Platform file → ④ Generic cookies.txt file

Common Questions

Why do I need cookies?
Some platforms (YouTube, Facebook, Instagram) block downloads of age-restricted, members-only, or private content unless you're authenticated. Cookies let the downloader act as your logged-in account.
Are my cookies safe?
Cookie files in the Cookies/ folder are stored only on this server. Using the Secrets method keeps them encrypted and out of your codebase entirely.
What format should the cookie file be?
Netscape HTTP Cookie format (the standard export format). Most browser extensions like "Get cookies.txt LOCALLY" export in this format by default.
Do cookies expire?
Yes. Browser cookies have expiry dates. If downloads start failing again, re-export fresh cookies from your browser and update the file or secret.
